# Development Principles

- Build working software first.
- Keep everything local by default.
- Every phase must be testable.
- Avoid premature optimization.
- Prioritize modularity.
- Security before convenience.
- Privacy before features.
- Quality before quantity.

---

# Rules

- Never skip a phase.
- Every phase must compile successfully.
- Every phase must have tests.
- Documentation must be updated before marking a phase complete.
- New features should not break previous phases.

---

# Definition of Done

A phase is complete when:

- All planned features are implemented.
- Tests pass.
- Documentation is updated.
- Performance is acceptable.
- No critical bugs remain.
- The application is usable before starting the next phase.
